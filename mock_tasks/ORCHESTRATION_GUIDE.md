# ClaudeMain オーケストレーションガイド

## 役割
ClaudeMainはモックアップ完成プロジェクトの総合調整役として、以下の責務を負います:

1. **タスク管理**: 各担当者の進捗確認と調整
2. **競合解決**: ファイル編集の競合を防止・解決
3. **品質管理**: 完成したモックアップの品質確認
4. **統合作業**: 共通コンポーネントの調整
5. **コミュニケーション**: チーム間の連絡調整

## 日次チェックリスト

### 朝（作業開始時）
- [ ] 各担当者の進捗ファイルを確認
  - `mock_tasks/progress/ClaudeSub01_progress.md`
  - `mock_tasks/progress/ClaudeSub02_progress.md`
  - `mock_tasks/progress/Codex01_progress.md`
- [ ] 競合の可能性がないか確認
- [ ] 前日の問題・質問に対応

### 夕方（作業終了時）
- [ ] 完了タスクの品質チェック
- [ ] TypeScriptビルドエラーチェック
- [ ] 全体進捗の更新
- [ ] 翌日のタスク優先順位調整

## 品質チェック手順

### 各画面の確認項目

```bash
# 1. TypeScriptエラーチェック
cd mio_desgin_system/ui_mockups_react/connecten-ui
npm run build

# 2. 開発サーバー起動
npm run dev

# 3. ブラウザで各画面確認
```

### チェックポイント
- [ ] TypeScriptコンパイルエラーなし
- [ ] コンソールエラーなし
- [ ] 画面が正しく表示される
- [ ] レスポンシブデザイン確認（モバイル、タブレット、デスクトップ）

## 進捗トラッキング

### 全体進捗計算式
```
総進捗率 = (完了画面数 / 総画面数) × 100
         = (完了画面数 / 88) × 100
```

### 担当者別進捗
- **ClaudeSub01**: 20画面 / 62画面残 (32%)
- **ClaudeSub02**: 18画面 / 62画面残 (29%)
- **Codex01**: 24画面 / 62画面残 (39%)

### 週次進捗レポート作成

毎週金曜日に以下を作成:
```markdown

- 完成画面数: XX / 88 (XX%)
- 今週完成: XX画面

- 進捗: XX%

### ClaudeSub02
- 完成: XX / 18
- 進捗: XX%

- 進捗: XX%
```

5. **全画面で動作確認**

## タスク再割り当て基準

### 再割り当てが必要な場合
- 進捗が大幅に遅れている
- 担当者が技術的困難に直面
- 作業量の不均衡が発覚

### 再割り当て手順
1. 現状分析
2. 新しい割り当て案作成
3. 各担当者に確認
4. 指示書更新
5. 作業再開

# ConnectEn - ダンススタジオ向けSaaS ERPプラットフォーム

## プロジェクト概要

ConnectEnは、ダンススタジオ業界に特化したマルチテナント型SaaS ERPプラットフォームです。補助金申請の承認を受け、エン株式会社のEn Dance Studioだけでなく、複数のダンススタジオが同一のプラットフォーム上で独立した環境を利用できる包括的なソリューションとして開発されています。

**コア機能の明確化**:
- **最優先**: スタジオ運営の核心機能（生徒管理・出席管理・支払い管理・レッスン管理）
- **補助機能**: ホットペッパービューティー型のマーケットプレイス機能

Mio Systemが手掛けるこのプロジェクトでは、スタジオ運営の効率化を第一に、最新のAI技術を活用した業界DXを推進します。

## 目的

### 最優先目標（核心機能）
- **生徒管理**: 登録・情報管理・出席率追跡・進捗管理の効率化
- **出席管理**: リアルタイム出席確認・出席率分析・アラート機能
- **支払い管理**: 自動請求・入金管理・滞納者検出・督促の自動化
- **レッスン・スケジュール管理**: クラス配属・講師アサイン・スタジオ稼働最適化

### 基盤機能
- ダンススタジオ業界向けのSaaSプラットフォームとして、マルチテナント対応のERPシステムを構築
- 既存の日本のシステムをベースに、スケーラブルなクラウドネイティブアーキテクチャで再設計
- アメリカの「MindBody」システムの優れた機能を取り入れ、グローバル標準の機能を提供

### 補助目標
- ホットペッパービューティー型の集客・予約マーケットプレイス機能を補助的に提供
- AI技術を活用し、各スタジオの業務効率化と顧客体験の向上を支援
- 業界全体のデータを匿名で集約・分析し、ベンチマークとインサイトを提供

## 主要機能

### 1. スタジオ運営核心機能（最優先）

#### 生徒管理
- 生徒登録・編集・プロフィール管理
- 生徒種別管理（本会員/バーチャル会員/体験会員）
- 生徒ステータス管理（アクティブ/休会/退会）
- 家族アカウント連携機能

#### 出席管理
- リアルタイム出席確認システム
- 出席・欠席・遅刻・公欠の詳細管理
- 生徒別・クラス別出席率分析
- 出席率低下アラート機能

#### 支払い管理
- マルチ決済対応（クレジットカード・口座振替・現金）
- 自動請求・引き落とし処理
- 滞納者自動検出・督促管理
- 入金確認・消込処理

#### レッスン・スケジュール管理
- レッスンスケジュール作成・管理
- 講師アサイン・配置最適化
- クラス配属管理（生徒レベル・希望考慮）
- マルチスタジオ統合予約システム

### 2. プラットフォーム機能
- マルチテナント管理（テナント登録、認証、データ分離）
- テナントオンボーディングとセルフサービスポータル
- 共通課金・請求管理
- プラットフォーム全体の分析・ベンチマーク

### 3. 補助機能

#### マーケットプレイス機能
- **スタジオ検索・発見機能**: エリア、ジャンル、価格帯での検索
- **統合予約システム**: 複数スタジオの横断的な予約管理
- **口コミ・レビュー機能**: 顧客によるスタジオ評価システム
- **クーポン・キャンペーン管理**: 新規顧客獲得のためのプロモーション機能
- **アフィリエイトネットワーク**: パートナーサイトとの連携
- **SEO最適化**: Google検索での上位表示支援

#### その他運営支援機能
- スタッフ・講師管理
- 在庫・物品管理
- マーケティング・コミュニケーション
- レポート・分析
- AI による業務最適化提案
- 在庫・物品管理
- モバイル対応
- リード管理・セールスパイプライン

## 技術スタック

- フロントエンド: Next.js, React, TypeScript, Tailwind CSS
- バックエンド: Node.js, Express, GraphQL
- データベース: PostgreSQL（マルチテナント対応）, Redis
- ORM: Prisma
- 認証: OAuth 2.0, JWT（テナント分離対応）
- インフラ: AWS (ECS/EKS, Lambda, CloudFront)
- デプロイ: Vercel（フロントエンド）, AWS（バックエンド）
- AI/ML: OpenAI API, AWS Comprehend, Hugging Face

## 開発者向け情報

### ローカル開発環境のセットアップ

```bash
# リポジトリをクローン
git clone [repository-url]

# アプリケーションディレクトリに移動
cd connect-en-app

# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev
```

### マルチテナント開発の注意点

- 全てのAPIエンドポイントはテナントIDを含む必要があります
- テナント間のデータ分離を徹底してください
- テナント固有の設定は環境変数ではなくデータベースに保存します

詳細な開発ガイドラインは[CLAUDE.md](./CLAUDE.md)を参照してください。
